Hãy đóng vai một senior software architect + full-stack engineer + product designer.
Tôi muốn xây dựng một web app hỗ trợ tạo “graph luật” để phục vụ việc hệ thống hóa và tra cứu văn bản pháp luật.
Yêu cầu chính:
- Giao diện dạng canvas/graph editor.
- Khi nhấp chuột phải lên canvas, tạo một node dạng bubble.
- Mỗi node có thể kéo thả, đổi tên, sửa nội dung, xóa.
- Mỗi node có thể nối với node khác bằng các cạnh (edges).
- Cho phép chọn loại quan hệ giữa các node, ví dụ: viện dẫn, sửa đổi, thay thế, căn cứ vào, hướng dẫn, liên quan đến.
- Hiển thị graph rõ ràng, trực quan, dễ mở rộng.
- Hỗ trợ zoom, pan, undo/redo, tìm kiếm node, highlight đường nối.
- Có thể lưu graph, tải lại graph, và xuất dữ liệu ra JSON/PDF/PNG.
Mục tiêu sử dụng:
- Hỗ trợ xây dựng cây/graph pháp luật.
- Quản lý quan hệ giữa luật, nghị định, thông tư, điều khoản, khoản, điểm.
- Dễ dùng cho người không rành kỹ thuật.
Yêu cầu kiến trúc:
- Hãy đề xuất kiến trúc tổng thể phù hợp nhất cho app này.
- Nêu rõ frontend, backend, database, storage, state management, rendering engine.
- Ưu tiên giải pháp dễ mở rộng, hiệu năng tốt, và có thể xử lý graph lớn.
- Nếu cần, hãy đề xuất cả kiến trúc module, event-driven, DDD, CQRS, hoặc microservice/hybrid phù hợp với quy mô.
Đầu ra mong muốn:
1. Mô tả ý tưởng sản phẩm.
2. Kiến trúc hệ thống đề xuất.
3. Danh sách tính năng MVP.
4. Data model cho node/edge/graph.
5. Flow người dùng.
6. Gợi ý tech stack.
7. Gợi ý cách triển khai UI/UX.
8. Nếu được, hãy viết luôn skeleton code hoặc cấu trúc thư mục ban đầu.
